NOTE 2: FakeProvider's synthetic path per symbol depends on Python's per-process hash randomization (`hash(symbol) % 1000` seeds its RNG), so anything sensitive to WHICH regime/value a given FakeProvider symbol produces (e.g. quant.regime_gate.classify_regime) is not stable across process runs — tests that call step() but aren't testing regime behavior specifically must monkeypatch `orch._regime_gate` to force a fixed regime, the same way they already monkeypatch `.analyze()`. ## IRON RULES
1. PAPER ONLY. No real broker execution without an explicit, separate, owner-confirmed phase.
2. RiskEngine veto is sacred: nothing may execute around it; the broker's approved-stamp check stays.
3. Every action gets an AuditLog record with reasoning. If it isn't audited, it didn't happen.
4. No fake AI: LLMOrchestrator only runs with a real key; suggestions must trace to computed numbers.
5. Secrets via .env/os.getenv only. Never commit .env. Never print keys.
6. LSE contract is VERIFIED from their official SDK (github.com/londonstrategicedge/lse-data v0.14.0): REST GET {vault}/candles with x-api-key header + custom User-Agent (their CDN blocks default python UA); /options/chain carries precomputed greeks. Yahoo fallback stays forever.
7. Tests before commit; small commits; tell the owner what to check after each push; never break main.
8. 1GB-class hosting: no torch/GPU deps; cache aggressively; every network call fails gracefully.
9. Honesty in UI copy: no promised returns; paper results labeled paper.
10. Strategy promotion gate (P7a, added 2026-07-17): a strategy's signals only convert into real (paper) entries once StrategyRegistry has promoted it PAPER — >= 30 settled forward-return signals AND a bootstrap 90% CI excluding zero. New strategies start in INCUBATION: logged, never traded. This NEVER blocks exits on an existing position — closing risk is always allowed regardless of promotion status.
11. Drawdown circuit breakers (P7e, added 2026-07-18): 5% drawdown from peak equity gradually cuts new-entry size toward 50%; 10% allows only risk-reducing trades; 15% is a sticky FULL STOP on new entries requiring DrawdownCircuitBreaker.manual_reset(reason) — a non-empty reason, audited — to clear. RiskEngine enforces this as a hard veto (defense-in-depth) independent of whatever sizing the orchestrator already applied. Existing positions can always be exited at any drawdown level; only new entries are ever restricted.
12. Transaction cost gate (P7b, added 2026-07-18): every proposed BUY carries an expected-cost estimate (spread + sqrt market impact) and an expected-edge figure; RiskEngine vetoes the entry if expected edge < 2x expected cost. Expected cost is shown on every proposed trade (BUY and SELL) in the audit trail even when it isn't gating the decision.
13. Regime-conditional trading (P7c, added 2026-07-18): quant.regime_gate classifies Bull/Bear/Storm from the P2 HMM detector and gates every new BUY entry — Storm forbids new trades outright and tightens the fallback stop on existing positions; Bear allows only dip-buy-at-extremes entries (urgency "🟡 FAST SETUP") at half size; Bull is unrestricted, full size. This stacks with (does not replace) the P7a promotion gate, P7e circuit breaker, and P7b cost gate — all four must clear for a new entry to execute. Exits are never gated by regime.
14. Correlation regime monitor (P7f, added 2026-07-19): quant.correlation_monitor tracks the ROLLING 20-day average pairwise correlation across live holdings (not the v0.3-era static full-history correlation_heat, which is unchanged and still runs alongside it). A hard ALERT at >=0.7 blocks new entries entirely; an early-warning state (rising trend, not yet at threshold) cuts new-entry size to 50% — "de-risk before the drawdown", not after. Stacks with the other four gates. Exits are never gated.
15. Portfolio stress test risk budget (P7g, added 2026-07-20): RuleOrchestrator.stress_test() runs a 10,000-path correlated Monte Carlo simulation of the live book (Ledoit-Wolf covariance — real diversification effects, not independent per-symbol sims) and caches a risk_budget in state.portfolio_stress; step() reads that cache every cycle and halves new-entry size when P(10% drawdown next month) >= 25%, until the next stress-test run replaces it. On-demand (LAB tab button) — no scheduler exists yet for literal weekly automation. Exits are never gated.
